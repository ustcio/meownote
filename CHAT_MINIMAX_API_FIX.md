#### 1.2 重构 getAIResponse 函数（第 225-249 行）
```typescript
async function getAIResponse(message: string): Promise<string> {
  try {
    const response = await fetch(`${API_BASE}/api/chat`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        message,
        model: currentModel,
        history: currentSession?.messages.map(m => ({
          role: m.role === 'user' ? 'user' : 'assistant',
          content: m.content
        })) || []
      })
    });

    if (!response.ok) {
      const errorData = await response.json().catch(() => ({}));
      throw new Error(errorData.message || `API error: ${response.status}`);
    }

    const data = await response.json();
    
    if (data.success && data.reply) {
      return data.reply;
    }
    
    if (data.error || data.message) {
      throw new Error(data.message || data.error || 'Unknown error');
    }
    
    throw new Error('Invalid response format from API');
  } catch (error) {
    console.error('API call failed:', error);
    throw error;
  }
}
```

### 常见问题排查

#### 1. "API not configured" 错误
**原因**：未在 Cloudflare Workers 环境变量中配置 `MINIMAX_API_KEY`

**解决方案**：
- 登录 Cloudflare Dashboard
- 进入 Workers & Pages
- 选择你的 Worker
- 在 Settings > Variables 中添加 `MINIMAX_API_KEY`

#### 2. "Authentication failed" 错误
**原因**：API 密钥无效或已过期

**解决方案**：
- 检查 API 密钥是否正确
- 确认密钥是否还有额度
- 在 MiniMax 平台查看使用情况

#### 3. 连接超时
**原因**：网络问题或 MiniMax 服务不可用

**解决方案**：
- 检查网络连接
- 查看 MiniMax 服务状态
- 等待重试
